The Primer

Welcome to our web development primer! This section of the site provides a step-by-step tutorial designed to teach you the fundamentals of state-of-the-art web development. The lessons proceed linearly, each building on the previous lesson, so I recommend that you do them in order. The table of contents is on this page. Just start at the top.

In these lessons you'll learn about the four major concerns associated with websites and web applications. These are:

  • Content
  • Structure (metadata)
  • Style (look & feel)
  • Behavior (interactivity)

Content

Our content will include not only the page text, but also images, videos, audio files, applets, etc. The textual content will be included in the page itself, surrounded by HTML, the other content will usually be stored separately on the server. We'll add HTML to our page that will tell the browser where to find this content and where to place it in the page.

Already you've learned something! Turns out web pages are not monlithic objects. They are actually more a structural framework upon which we hang some text, and then insert the images, etc. as they are loaded.

Structure

Web pages aren't just text and images. Surrounding this content is an invisible structure—invisible to the user, that is—that instructs the browser what to do with each item of content. More than that, the structure provides semantic information about the content. We sometimes call this metadata because we like big words.

Meta means above or beyond or about. So metadata is data about data. This metadata is typically invisible to the end user, but it allows the browser to understand, on some very basic, not-too-bright, computer level what the content means.

To create this structure, we use a markup language. A markup language is exactly what it says it is: a language used to mark up content. In a sense, all we're doing is labelling the content so the browser can recognize it. For this purpose, we'll use the world's most popular markup language: the HyperText Markup Language, better known as HTML.

And, of course, we'll be using the latest and greatest version of HTML, known and loved by all as HTML5. Because you deserve the best. Don't you?

Style

Life would be rather boring without style. But there's more to it than just looking good. A properly-styled web page is a more usable (user friendly) web page. In other words, style has an important role in functionality. It's more than just a pretty face.

In this course, we'll do 100% of our styling with Cascading Style Sheets or CSS. We'll be spending a lot of time on version 3 of the CSS standard, known colloquially as CSS3, but as CSS3 builds on CSS2 and CSS1, we'll be covering plenty of those as well.

Behavior

As you may have noticed, wep pages are capable of much more than just displaying text and images. A well-designed web page is an interactive, multimedia experience. The key word here is interactive. It is interactivity that has made the World Wide Web the ubiquitous information source it has become. We've one-upped TV.

Most interactivity in browsers is provided by scripts, and the scripting language of the World Wide Web is JavaScript. So we'll be learning a bit of that. But much of what we need to add interactivity to our pages is provided by various JavaScript libraries. These are nothing more than re-usable code written by others to save us time and effort. And we like saving time and effort, right?

The most popular of these libraries—currently, at least— is jQuery. Guess what we'll be learning more than a little of? If you guessed, jQuery, you catch on quickly.

But wait! There's more...

The biggest mistake that most beginner web developers make is to mix the above concerns. In this course, you'll hear me speak—so to speak—about a key principle of programming called, separation of concerns. This is the key to good website design.

We'll also get into two very important subjects in some depth: usability and accessibility to persons with disabilities. These are vitally important aspects of web development that are often overlooked by new developers (and, sadly, by many who really ought to know better).

Finally, we'll cover various bits and pieces that will help you to understand better how it all fits together: the OSI model, a little bit about the HyperText Transfer Protocol (HTTP), the basics of REpresentational State Transfer (REST), and more.

So what are you waiting for! Jump right in!
